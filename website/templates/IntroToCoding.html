Object-orientated Programming

R is an object-oriented coding language (also know as object-oriented programming, OOP). To be completely honest, understanding what object-oriented coding is, is a concept that MOST people will struggle with. You will likely not understand it right away. I tried to write this tutorial in such a way that by the end you will have a solid grasp of it; however, it will likely be hard to grasp and you might have to do a few of the modules several times to understand. Basically, OOP is a framework of coding in which a programmer (that’s you) assigns datasets, values, lists, vectors, functions, etc. into things called objects. In R an object is just a letter or word that stores information. Objects names are arbitrary and do not need to be tied to the information they are storing (although good coders will connect the name to the information so it is easier to remember later).

The following list are all possible object names:

a
apple
data
x
Time4food
I.dont.remember
Supercalifragilisticexpialidocious

** It is extemely important to know that R is case sensitive. That means the apple is different than Apple. This is a big source of frustration for coders new and old alike.

Again, the names do no have to be meaningful (but most good coders will make them meaningful). In R, you assign things into objects an <- also know as the assinment operator. Some coders also use the =. However, I recommend using this only inside of functions and use the <- while assigning and object. More on that later.

In the following chuck of code, I have assigned the number 1 into an object I have named x. Then I have told R to print whatever x is (line 2). While going through this tutorial, the dark grey boxes represent input (or code I have written) while the white boxes are the corresponding output from those lines of code (or response that R gives me). This is a rather elementary example. Since I set x to the number 1, I didn’t need to print it but it highlights the fact that by default R will not display an object in the console unless you tell it to.

x <- 1
x
## [1] 1

**If you are following along in a script, then you will notice that the object x has been added to your environment.

One final important thing to note about code creation is that it is read in the order that you run it. When running multiple lines of code at the same time that means

x <- 1
x <- 5
x
## [1] 5

** Importantly, if you highlight code and run it out of order then the objects will also be created out of order, which can lead to disastrous consequences.

As you see above, the output was 5. Initially R made a object called x, which was 1. However, on line 2 I assigned x as 5. Therefore, R overwrote what x used to be and recreated x as my newest assignment.

Functions

In general, a function is a tool that a programmer can use in order to more quickly and efficiently obtain some result. For example, a researcher might want to know what the mean is for a variable, so they would use the mean function. In R, you can tell that something is a function because it will have parentheses like so mean(). Importantly, there must not be a space between the name of a function and the opening parentheses. Inside of functions you include what are called arguments. For example, you would include the object name of the variable that you want to find the mean of inside the parentheses mean(x). In the following line of code, I will use the concatenate function c() to create an object called age. This will represent the ages of 5 individuals, so we can get the mean age of these people.

age <- c(19,19,20,18,18)
mean(age)
## [1] 18.8

Functions follow the stucture above where you have the names of the function. Inside the parenthese you have the arguments, or inputs, of the function. If you have multiple arguments, you will need to seperate them with commas like in the concatenate function above. Each age was seperated by a comma.

Module XX: For Loops

library("MASS")
#Creating a sampling distribution
N <- 10000 # Population size
n <- 25 # Sample size
mu <- 50 # Popultaion Mean
sigma <- 10 # Population Standard deviation
x <- rnorm(N, mu, sigma) # Create the population
mdraws <- matrix(0, 10000, 1) # Create an empty matrix to be filled in by the For Loop
for(i in 1:10000){ # Starts the For Loop with i as the iterator and loops 10,000 times
  draw1 <- sample(x, size = n, replace = TRUE) # Draws a sample from the population
  mdraws[i] <- mean(draw1) # Take the mean of that sample and puts it into our empty matrix
}

hist(mdraws, breaks = 100, col='#5900b3') # Plot the samples as a histogram

Basic Plots

The most basic plots can be created with the plot() function. In the following example I will make two correlated variables and plot the scatter plot.

x <- rnorm(20, 0, 1)
y <- rnorm(20, 0, 1)*(.8*x)
plot(x, y)

Sometimes you might need to make a histogram. As you might expect, that can be done with the hist() function.

x <- rnorm(1000, 0, 1)
hist(x)

Although these plots were easily created, they are not very asethetically appealling, as you might have noticed. As you might expect, there are arguments, that can adjust the color, size, line type, title, axis title, etc. However, even with those adjustments you would not be using the full plotting power of R. In order to get the most out of your plots in R, you will need to install the ggplot2 package. More on ggplot2 in ‘Advanced Plotting.’
